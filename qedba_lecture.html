1 Adversarial Attacks: Making Models Misled

In one sentence: An adversarial attack adds a small, carefully chosen change to an input so the model gives the wrong answer.

Red teaming means deliberately attacking models to find their weaknesses — the same way security teams attack their own systems. Understanding attacks is how we learn to build more robust models.

A poisoning attack spoofs the model: it perturbs the input so the model is misled. A real-world example from the lecture: an adversarial attack could make a model read a stop sign as a speed limit sign — a small change to the input, dangerous consequences.

The change is small enough that a human still sees a stop sign. The model, however, is confidently wrong.

2 Threat Models: What Does the Attacker Know?

In one sentence: Attacks are categorized by how much access the attacker has — from full model access down to label-only.
White-Box

Full Access

The attacker sees the model's weights, gradients, and confidence scores. Easiest to attack — just follow the gradient.

Score-Based (Grey Box)

Scores Only

The attacker sees confidence scores ("cat: 0.92, dog: 0.08") but not the model's internals. The scores still leak useful hints.

Decision-Based (DBA)

Label Only

The attacker only gets "cat" or "dog" back. No scores, no gradients. The most realistic — and hardest — setting, e.g. a cloud API.

Decision-based attacks: poison the input by observing the output

In a DBA, the attacker repeatedly queries the model and watches only the labels that come back. The key question: can we craft a poisoned input using nothing but those labels?

The attacker also wants to minimize the budget — use as few queries and resources as possible. Fewer queries means cheaper, faster, and harder to detect: real APIs limit query rates to thwart suspicious behavior.

3 Loss Functions & Gradient Descent

In one sentence: Models learn by finding the lowest point of a "wrongness score" — by always stepping downhill.

First, the basics: what is a loss function?

A loss function (also called a cost function) is just a score for how wrong the model is. Show the model a cat photo: if it says "cat" with high confidence, the loss is small; if it says "dog", the loss is large. One number = how bad the model's answer was.

Why find the lowest point? Training a model means adjusting its weights to make this wrongness score as small as possible. The lowest point of the loss function is where the model makes the fewest mistakes — so "learning" is literally a search for the bottom of the curve. As Dr. Yu said: to train a model, you need to know how to define its cost function.

In attacks, the same idea is flipped: the attacker defines their own loss — for example, "how far is this image from being misclassified?" — and searches for its lowest point, where the model gets fooled.

Imagine hiking down a hill in thick fog. You can't see the valley, but you can feel which way the ground slopes — so you keep stepping downhill. The gradient is that slope: it points uphill, so you step the opposite way. The step size is the learning rate. Repeat until you reach a minimum. This is gradient descent — how neural networks learn, and how white-box attacks work.

4 Why Black-Box Attacks Are Hard

In one sentence: Without gradients, you must estimate them by querying — and that costs tens of thousands of queries.

Zeroth-order gradient descent

Gradient descent needs the actual gradient at each step. In a black-box setting, we cannot calculate it — we have to estimate it by sampling many nearby points and observing which way the output changes. This estimate-by-querying approach is called zeroth-order optimization. Each sample costs a query — which is why these attacks need 20,000+ queries.

The way out: gradient-free optimization

What if we skip gradients entirely? Gradient-free optimization treats the attack as a pure search problem. The smartest version of this is Bayesian Optimization, built on a tool called the Gaussian Process — the next two sections introduce both.

5 Gaussian Process (GP)

In one sentence: A GP draws a best-guess curve through the points you've observed — and honestly marks how unsure it is everywhere else.

A Gaussian Process is a way to model an unknown function from only a few observations. Think of sketching a smooth curve through a handful of known dots: between the dots you're guessing, and a GP keeps track of exactly how uncertain each guess is.

6 Bayesian Optimization (BO)

In one sentence: Use the GP's predictions and uncertainty to pick the single most useful point to query next.

Bayesian Optimization uses a Gaussian Process to decide where to query next. Instead of querying randomly or estimating gradients, it asks: "Where is the next query most likely to pay off?"

It's like choosing a restaurant: do you go back to your known favorite, or try a new place that might be even better? BO uses a simple scoring rule (called the acquisition function) to balance this trade-off:

• Exploitation — query near the current best known point.
• Exploration — query in uncertain regions where there might be something better.

This balance is what makes BO so query-efficient: no query is wasted on regions that are neither promising nor informative.
